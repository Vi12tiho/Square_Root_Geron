Проделать и продемонстрировать указанную последовательность действий, воспользовавшись системой контроля версий git:

Нечетный вариант.

1. Зарегистрироваться на github.
2. Создать локально на компьютере git-репозиторий, добавив туда один из реализованных ранее проектов (например, проект по работе с БД).
3. Изменить интерфейс программы.
4. Выполнить коммит.
5. Записать репозиторий на сервер.
6. Создать новую ветку child (локально на Вашем компьютере и на основе ветки main).
7. Переключиться на ветку child.
8. Добавить в проект 3 файла jpeg, 
9. Выполнить коммит.
10. Создать новую ветку superchild (локально на Вашем компьютере и на основе ветки child).
11. Переключиться на ветку superchild .
12. Добавить в файл readme.txt новую информацию.
13. Добавить новый файл jpeg.
14. Выполнить коммит.
15. Переключиться на ветку child.
16. Удалить два файла jpeg.
17. Провести слияние веток child и superchild.
18. Добавить и наполнить данными файл help.txt.
19. Выполнить коммит.
20. Провести слияние веток child и master.
21. Удалить ветку child.
22. Вывести информацию о всех коммитах.
